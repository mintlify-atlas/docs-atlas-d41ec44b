## Overview

torch_brain uses PyTorch Lightning to provide a scalable framework for training neural population decoders. This guide covers training both POYO (single-task) and POYO+ (multi-task) models using real neural data.

## Quick Start

<Steps>
  <Step title="Install dependencies">
    Install torch_brain and required packages:
    ```bash
    pip install pytorch_brain lightning wandb brainsets
    ```
  </Step>

  <Step title="Download data">
    Use brainsets to download training data:
    ```bash
    brainsets prepare perich_miller_population_2018
    ```
  </Step>

  <Step title="Run training">
    Start training with a config file:
    ```bash
    python train.py --config-name train_poyo_mp.yaml
    ```
  </Step>
</Steps>

## Training Architecture

### Main Components

The training pipeline consists of three core components:

1. **TrainWrapper** - Lightning module that handles training/validation loops
2. **DataModule** - Manages data loading and preprocessing
3. **Model** - The POYO or POYO+ architecture

### Training Script Structure

Here's the basic structure from `examples/poyo/train.py`:

```python
import hydra
import lightning as L
from torch_brain.models.poyo import POYO
from torch_brain.optim import SparseLamb

class TrainWrapper(L.LightningModule):
    def __init__(self, cfg, model, modality_spec):
        super().__init__()
        self.cfg = cfg
        self.model = model
        self.modality_spec = modality_spec
        self.save_hyperparameters(OmegaConf.to_container(cfg))

    def training_step(self, batch, batch_idx):
        # Forward pass
        output_values = self.model(**batch["model_inputs"])
        
        # Compute loss
        mask = batch["model_inputs"]["output_mask"]
        output_values = output_values[mask]
        target_values = batch["target_values"][mask]
        target_weights = batch["target_weights"][mask]
        
        loss = self.modality_spec.loss_fn(
            output_values, target_values, target_weights
        )
        
        self.log("train_loss", loss, prog_bar=True)
        return loss
```

## Optimizer Configuration

### SparseLamb Optimizer

torch_brain uses SparseLamb for efficient training with sparse embeddings:

```python
def configure_optimizers(self):
    # Linear learning rate scaling
    max_lr = self.cfg.optim.base_lr * self.cfg.batch_size
    
    # Separate parameter groups for sparse embeddings
    special_emb_params = (
        list(self.model.unit_emb.parameters()) + 
        list(self.model.session_emb.parameters())
    )
    
    remaining_params = [
        p for n, p in self.model.named_parameters()
        if "unit_emb" not in n and "session_emb" not in n
    ]
    
    optimizer = SparseLamb(
        [
            {"params": special_emb_params, "sparse": True},
            {"params": remaining_params},
        ],
        lr=max_lr,
        weight_decay=self.cfg.optim.weight_decay,
    )
    
    # OneCycleLR scheduler
    scheduler = torch.optim.lr_scheduler.OneCycleLR(
        optimizer,
        max_lr=max_lr,
        total_steps=self.trainer.estimated_stepping_batches,
        pct_start=self.cfg.optim.lr_decay_start,
        anneal_strategy="cos",
        div_factor=1,
    )
    
    return {
        "optimizer": optimizer,
        "lr_scheduler": {
            "scheduler": scheduler,
            "interval": "step",
        },
    }
```

From `examples/poyo/train.py:53-90`.

<Tip>
  The linear scaling rule (`max_lr = base_lr * batch_size`) ensures stable training when changing batch sizes.
</Tip>

## Data Loading

### DataModule Setup

The DataModule links your dataset with the model:

```python
class DataModule(L.LightningDataModule):
    def __init__(self, cfg):
        super().__init__()
        self.cfg = cfg
        
        # Create train and eval datasets
        train_transforms = hydra.utils.instantiate(cfg.train_transforms)
        self.train_dataset = hydra.utils.instantiate(
            cfg.dataset,
            root=cfg.data_root,
            transform=Compose([*train_transforms]),
        )
        
        eval_transforms = hydra.utils.instantiate(cfg.eval_transforms)
        self.eval_dataset = hydra.utils.instantiate(
            cfg.dataset,
            root=cfg.data_root,
            transform=Compose([*eval_transforms]),
        )
    
    def link_model(self, model):
        """Setup datasets and initialize model vocabularies"""
        self.sequence_length = model.sequence_length
        self.train_dataset.transform.transforms.append(model.tokenize)
        self.eval_dataset.transform.transforms.append(model.tokenize)
        self._init_model_vocab(model)
    
    def _init_model_vocab(self, model):
        model.unit_emb.initialize_vocab(self.get_unit_ids())
        model.session_emb.initialize_vocab(self.get_session_ids())
```

## Training Execution

### Main Training Loop

The main function orchestrates the entire training process:

```python
@hydra.main(version_base="1.3", config_path="./configs", config_name="train.yaml")
def main(cfg):
    # Set random seed
    seed_everything(cfg.seed)
    
    # Setup logging
    wandb_logger = None
    if cfg.wandb.enable:
        wandb_logger = L.pytorch.loggers.WandbLogger(
            save_dir=cfg.log_dir,
            entity=cfg.wandb.entity,
            name=cfg.wandb.run_name,
            project=cfg.wandb.project,
            log_model=cfg.wandb.log_model,
        )
    
    # Create model and data
    data_module = DataModule(cfg=cfg)
    model = hydra.utils.instantiate(cfg.model, readout_spec=readout_spec)
    data_module.link_model(model)
    
    # Create training wrapper
    wrapper = TrainWrapper(cfg=cfg, model=model, modality_spec=readout_spec)
    
    # Setup trainer
    trainer = L.Trainer(
        logger=wandb_logger,
        default_root_dir=cfg.log_dir,
        check_val_every_n_epoch=cfg.eval_epochs,
        max_epochs=cfg.epochs,
        callbacks=callbacks,
        precision=cfg.precision,
        accelerator="gpu" if torch.cuda.is_available() else "cpu",
        devices=cfg.gpus,
        num_nodes=cfg.nodes,
    )
    
    # Train and test
    trainer.fit(wrapper, data_module, ckpt_path=cfg.ckpt_path)
    trainer.test(wrapper, data_module, ckpt_path="best", weights_only=False)
```

## Multi-Task Training (POYO+)

### Key Differences

POYO+ handles multiple readout tasks simultaneously. The main changes are:

1. **Multi-task loss computation**:

```python
def training_step(self, batch, batch_idx):
    output_values = self.model(**batch["model_inputs"], unpack_output=False)
    
    loss = torch.tensor(0, device=self.device, dtype=torch.float32)
    taskwise_loss = {}
    
    for readout_id in output_values.keys():
        output = output_values[readout_id]
        target = target_values[readout_id]
        spec = self.model.readout.readout_specs[readout_id]
        
        weights = target_weights.get(readout_id, 1.0)
        taskwise_loss[readout_id] = spec.loss_fn(output, target, weights)
        
        # Weight by number of sequences with this task
        num_sequences = torch.any(
            batch["model_inputs"]["output_decoder_index"] == spec.id,
            dim=1,
        ).sum()
        loss += taskwise_loss[readout_id] * num_sequences
    
    loss = loss / batch_size
    self.log("train_loss", loss, prog_bar=True)
    self.log_dict({f"losses/{k}": v for k, v in taskwise_loss.items()})
    return loss
```

From `examples/poyo_plus/train.py:95-146`.

## Troubleshooting

<AccordionGroup>
  <Accordion title="Out of Memory Errors">
    **Solutions**:
    - Reduce `batch_size` in config
    - Decrease `sequence_length` (process shorter time windows)
    - Use gradient accumulation:
      ```yaml
      accumulate_grad_batches: 4
      ```
    - Enable mixed precision:
      ```yaml
      precision: 16-mixed
      ```
  </Accordion>

  <Accordion title="NaN Loss">
    **Common causes**:
    - Learning rate too high - try reducing `base_lr`
    - Missing data normalization - check your transforms
    - Gradient explosion - enable gradient clipping:
      ```python
      trainer = L.Trainer(
          gradient_clip_val=1.0,
          gradient_clip_algorithm="norm",
      )
      ```
  </Accordion>

  <Accordion title="Slow Training">
    **Optimization tips**:
    - Increase `num_workers` for data loading
    - Use `pin_memory=True` (already default)
    - Enable tensor cores:
      ```python
      torch.set_float32_matmul_precision("medium")
      ```
    - Check `prefetch_factor` setting
    - Monitor with `EpochTimeLogger` callback
  </Accordion>

  <Accordion title="Validation Hanging">
    **Possible issues**:
    - Set `limit_val_batches=None` to ensure all validation data is processed
    - Check sampler configuration
    - Disable sanity checks during debugging:
      ```yaml
      sanity_check_validation: false
      ```
    - Reduce `num_workers` to 0 for validation dataloaders to simplify debugging
  </Accordion>

  <Accordion title="Checkpoint Not Saving">
    **Verify**:
    - `ModelCheckpoint` callback is included
    - `monitor` metric is being logged
    - `save_on_train_epoch_end=True` for epoch-based checkpointing
    - Check disk space and permissions in `log_dir`
  </Accordion>
</AccordionGroup>

## Performance Optimization

### Tensor Core Acceleration

Enable faster matrix operations on modern GPUs:

```python
torch.set_float32_matmul_precision("medium")
```

This is already included in the example scripts.

### DataLoader Tuning

Optimal settings for neural data:

```python
train_loader = DataLoader(
    dataset,
    batch_size=cfg.batch_size,
    num_workers=4,  # Adjust based on CPU cores
    pin_memory=True,  # Faster GPU transfer
```
