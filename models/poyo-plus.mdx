## Architecture

POYO+ extends POYO with multi-task capabilities through task-specific embeddings and readout layers:

1. **Input Token Construction**: Combines unit embeddings, token type embeddings, and time embeddings for each spike
2. **Compression**: Cross-attention compresses the input sequence into learnable latent tokens
3. **Processing**: Multiple self-attention layers refine the latent representations
4. **Multi-Task Query Construction**: Combines task embeddings, session embeddings, and output timestamps
5. **Decoding**: Cross-attention produces outputs in the model's dimensional space
6. **Multi-Task Readout**: Task-specific linear layers map to appropriate output dimensions for each task

### Key Differences from POYO

- **Task Embeddings**: Learnable embeddings for each prediction task
- **Multitask Readout**: Separate linear layers for each task instead of a single readout
- **Flexible Task Specification**: Configure multiple tasks via `readout_specs` dictionary
- **Shared Encoder**: All tasks share the same encoder and processor, enabling transfer learning

## Usage

### Basic Initialization

```python
from torch_brain.models import POYOPlus
from torch_brain.registry import MODALITY_REGISTRY

# Use pre-registered modalities or define custom ones
readout_specs = {
    "velocity": MODALITY_REGISTRY["velocity"],
    "hand_position": MODALITY_REGISTRY["hand_position"],
    # Add more tasks as needed
}

# Initialize POYO+
model = POYOPlus(
    sequence_length=1.0,  # 1 second windows
    readout_specs=readout_specs,  # Dict of task specs
    latent_step=0.125,  # latent tokens every 125ms
    num_latents_per_step=64,
    dim=512,
    depth=2,
    dim_head=64,
    cross_heads=1,
    self_heads=8,
)
```

### Using All Registered Modalities

```python
from torch_brain.registry import MODALITY_REGISTRY

# Use all available modalities
model = POYOPlus(
    sequence_length=1.0,
    readout_specs=MODALITY_REGISTRY,  # All registered tasks
    latent_step=0.125,
    # ... other params
)
```

### Initialize Vocabularies

```python
# Initialize unit and session vocabularies before using the model
unit_ids = dataset.get_unit_ids()
model.unit_emb.initialize_vocab(unit_ids)

session_ids = dataset.get_session_ids()
model.session_emb.initialize_vocab(session_ids)
```

### Forward Pass

```python
import torch

# Forward pass returns dict of outputs by task
outputs = model(
    # Input sequence (spikes)
    input_unit_index=unit_indices,
    input_timestamps=spike_times,
    input_token_type=token_types,
    input_mask=input_mask,
    # Latent sequence
    latent_index=latent_indices,
    latent_timestamps=latent_times,
    # Output queries
    output_session_index=session_indices,
    output_timestamps=output_times,
    output_decoder_index=task_indices,  # Which task for each query
    unpack_output=False,
)

# outputs is a dict: {task_name: tensor}
# e.g., {"velocity": tensor(...), "hand_position": tensor(...)}
```

### Multi-Task Training

```python
# Compute task-specific losses
total_loss = 0
for task_name, predictions in outputs.items():
    targets = batch["target_values"][task_name]
    weights = batch["target_weights"].get(task_name, 1.0)
    
    spec = model.readout.readout_specs[task_name]
    task_loss = spec.loss_fn(predictions, targets, weights)
    
    total_loss += task_loss

total_loss.backward()
```

### Using the Tokenizer

```python
from temporaldata import Data

# Tokenizer handles multi-task data preparation
tokenized = model.tokenize(data)

# Returns dict with:
# - model_inputs: includes output_decoder_index for task routing
# - target_values: dict of targets by task
# - target_weights: dict of weights by task
# - eval_mask: dict of eval masks by task
```

## Parameters

<ParamField path="sequence_length" type="float" required>
  Maximum duration of the input spike sequence in seconds
</ParamField>

<ParamField path="readout_specs" type="Dict[str, ModalitySpec]" default="MODALITY_REGISTRY">
  Dictionary mapping task names to `ModalitySpec` objects. Each spec defines the output dimension, loss function, and metrics for that task
</ParamField>

<ParamField path="latent_step" type="float" required>
  Timestep of the latent grid in seconds. Controls temporal resolution of compressed representation
</ParamField>

<ParamField path="num_latents_per_step" type="int" default="64">
  Number of unique latent tokens repeated at every latent step
</ParamField>

<ParamField path="dim" type="int" default="512">
  Hidden dimension of the model for all embeddings and layers
</ParamField>

<ParamField path="depth" type="int" default="2">
  Number of processing layers (self-attention blocks in latent space)
</ParamField>

<ParamField path="dim_head" type="int" default="64">
  Dimension of each attention head
</ParamField>

<ParamField path="cross_heads" type="int" default="1">
  Number of attention heads in cross-attention layers
</ParamField>

<ParamField path="self_heads" type="int" default="8">
  Number of attention heads in self-attention layers
</ParamField>

<ParamField path="ffn_dropout" type="float" default="0.2">
  Dropout rate for feed-forward networks
</ParamField>

<ParamField path="lin_dropout" type="float" default="0.4">
  Dropout rate for linear layers in processing blocks
</ParamField>

<ParamField path="atn_dropout" type="float" default="0.0">
  Dropout rate for attention layers
</ParamField>

<ParamField path="emb_init_scale" type="float" default="0.02">
  Scale for embedding initialization
</ParamField>

<ParamField path="t_min" type="float" default="1e-4">
  Minimum timestamp resolution for rotary embeddings (seconds)
</ParamField>

<ParamField path="t_max" type="float" default="2.0627">
  Maximum timestamp resolution for rotary embeddings (seconds)
</ParamField>

## Complete Training Example

See `torch_brain/models/poyo_plus.py:28` for the full implementation.

```python
import lightning as L
from torch.utils.data import DataLoader
from torch_brain.models import POYOPlus
from torch_brain.data import Dataset, collate
from torch_brain.registry import MODALITY_REGISTRY

# Initialize model with multiple tasks
model = POYOPlus(
    sequence_length=1.0,
    readout_specs=MODALITY_REGISTRY,
    latent_step=0.125,
    dim=512,
    depth=6,
    num_latents_per_step=64,
)

# Setup dataset and initialize vocabularies
dataset = Dataset(root="/path/to/data", split="train")
model.unit_emb.initialize_vocab(dataset.get_unit_ids())
model.session_emb.initialize_vocab(dataset.get_session_ids())

# Add tokenizer as transform
dataset.transform = model.tokenize

# Create dataloader
train_loader = DataLoader(
    dataset,
    batch_size=32,
    collate_fn=collate,
    num_workers=4,
)

# Train with Lightning (see examples/poyo_plus/train.py for full code)
trainer = L.Trainer(max_epochs=100, accelerator="gpu")
trainer.fit(model, train_loader)
```

## Key Features

### Task Embeddings

POYO+ learns a separate embedding for each task, allowing the model to route information appropriately:

```python
# Task embeddings are learned during training
# Located at: model.task_emb
num_tasks = len(readout_specs)
task_emb_layer = model.task_emb  # Embedding(num_tasks, dim)
```

### Multitask Readout Layer

Each task has its own linear projection from the model dimension to its specific output dimension:

```python
# Access task-specific readout layers
velocity_readout = model.readout.readout_layers["velocity"]
# Returns: Linear(dim -> velocity_dim)
```

### Shared Representations

The encoder and processor layers are shared across all tasks, enabling:
- Transfer learning between related tasks
- More efficient use of data
- Improved generalization through multi-task learning
