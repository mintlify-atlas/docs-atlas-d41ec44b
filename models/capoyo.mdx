CaPOYO (Calcium POYO+) extends the POYO+ architecture for calcium imaging recordings, adding a calcium value mapping layer to handle fluorescence traces.

## Architecture

CaPOYO adapts POYO+ for calcium imaging data:

1. **Input Preparation**: Calcium fluorescence traces (dF/F) are prepared for each neuron and timepoint
2. **Calcium Value Mapping**: A linear layer maps calcium values to half the model dimension
3. **Unit Embeddings**: Unit (neuron) embeddings are mapped to the other half of the model dimension
4. **Combined Input**: Calcium values and unit embeddings are concatenated
5. **Compression, Processing, Decoding**: Same as POYO+ with shared encoder/processor and multi-task readouts

### Key Differences from POYO+

- **Input Value Map**: Linear layer `input_value_map` processes continuous calcium fluorescence values
- **No Token Types**: Calcium imaging doesn't use spike token types (start/end/spike)
- **Dense Temporal Sampling**: Processes calcium traces at imaging frame rate instead of sparse spikes
- **Half-Dimensional Embeddings**: Unit embeddings are `dim // 2` to accommodate concatenation with value features

## Usage

### Basic Initialization

```python
from torch_brain.models import CaPOYO
from torch_brain.registry import MODALITY_REGISTRY

# Define tasks for calcium imaging experiments
readout_specs = {
    "stimulus_orientation": MODALITY_REGISTRY["stimulus_orientation"],
    "running_speed": MODALITY_REGISTRY["running_speed"],
    # Add more visual coding tasks
}

# Initialize CaPOYO
model = CaPOYO(
    sequence_length=2.0,  # 2 second windows
    readout_specs=readout_specs,
    latent_step=0.1,  # 100ms latent steps
    num_latents_per_step=64,
    dim=512,
    depth=4,
    dim_head=64,
    cross_heads=1,
    self_heads=8,
)
```

### Initialize Vocabularies

```python
# Initialize vocabularies before using the model
unit_ids = dataset.get_unit_ids()  # Neuron/ROI IDs
model.unit_emb.initialize_vocab(unit_ids)

session_ids = dataset.get_session_ids()
model.session_emb.initialize_vocab(session_ids)
```

### Forward Pass

```python
import torch

# Forward pass with calcium traces
outputs = model(
    # Input sequence (calcium traces)
    input_unit_index=neuron_indices,      # Which neuron
    input_timestamps=frame_times,         # Frame timestamps
    input_values=calcium_values,          # dF/F values (shape: batch, n_in, 1)
    input_mask=input_mask,
    # Latent sequence
    latent_index=latent_indices,
    latent_timestamps=latent_times,
    # Output queries
    output_session_index=session_indices,
    output_timestamps=output_times,
    output_decoder_index=task_indices,
    unpack_output=False,
)

# outputs: dict mapping task names to predictions
# e.g., {"stimulus_orientation": tensor(...), "running_speed": tensor(...)}
```

### Using the Tokenizer

```python
from temporaldata import Data

# Tokenizer converts calcium imaging Data to model inputs
tokenized = model.tokenize(data)

# Expects data.calcium_traces with:
# - timestamps: array of frame times
# - df_over_f: array of shape (T, N) for T timepoints and N neurons

# Returns dict with:
# - model_inputs: includes input_values for calcium traces
# - target_values: dict of targets by task
# - target_weights: dict of weights by task
# - eval_mask: dict of eval masks by task
```

## Parameters

<ParamField path="sequence_length" type="float" required>
  Maximum duration of the input sequence in seconds. For calcium imaging, this should match your recording window length
</ParamField>

<ParamField path="readout_specs" type="Dict[str, ModalitySpec]" default="MODALITY_REGISTRY">
  Dictionary mapping task names to `ModalitySpec` objects. Define all prediction tasks for your imaging experiment
</ParamField>

<ParamField path="latent_step" type="float" required>
  Timestep of the latent grid in seconds. Can be aligned with imaging frame rate or set independently
</ParamField>

<ParamField path="num_latents_per_step" type="int" default="64">
  Number of unique latent tokens repeated at every latent step
</ParamField>

<ParamField path="dim" type="int" default="512">
  Hidden dimension of the model. Note: unit embeddings will be `dim // 2` to concatenate with calcium value features
</ParamField>

<ParamField path="depth" type="int" default="2">
  Number of processing layers (self-attention blocks in latent space)
</ParamField>

<ParamField path="dim_head" type="int" default="64">
  Dimension of each attention head
</ParamField>

<ParamField path="cross_heads" type="int" default="1">
  Number of attention heads in cross-attention layers
</ParamField>

<ParamField path="self_heads" type="int" default="8">
  Number of attention heads in self-attention layers
</ParamField>

<ParamField path="ffn_dropout" type="float" default="0.2">
  Dropout rate for feed-forward networks
</ParamField>

<ParamField path="lin_dropout" type="float" default="0.4">
  Dropout rate for linear layers in processing blocks
</ParamField>

<ParamField path="atn_dropout" type="float" default="0.0">
  Dropout rate for attention layers
</ParamField>

<ParamField path="emb_init_scale" type="float" default="0.02">
  Scale for embedding initialization using truncated normal distribution
</ParamField>

<ParamField path="t_min" type="float" default="1e-4">
  Minimum timestamp resolution for rotary embeddings (seconds)
</ParamField>

<ParamField path="t_max" type="float" default="2.0627">
  Maximum timestamp resolution for rotary embeddings (seconds)
</ParamField>

## Training on Allen Visual Coding Dataset

CaPOYO was trained on the Allen Visual Coding Ophys dataset with 1304 sessions:

```python
import lightning as L
from torch_brain.models import CaPOYO
from torch_brain.data import Dataset, collate
from torch_brain.registry import MODALITY_REGISTRY

# Initialize model
model = CaPOYO(
    sequence_length=2.0,
    readout_specs=MODALITY_REGISTRY,
    latent_step=0.1,
    dim=512,
    depth=4,
)

# Setup dataset
dataset = Dataset(
    root="/path/to/allen_visual_coding_ophys_2016",
    split="train",
)

# Initialize vocabularies
model.unit_emb.initialize_vocab(dataset.get_unit_ids())
model.session_emb.initialize_vocab(dataset.get_session_ids())

# Add tokenizer
dataset.transform = model.tokenize

# Train (see examples/poyo_plus/train.py for full training loop)
```

## Architecture Details

### Calcium Value Mapping

The input value map transforms continuous calcium traces:

```python
# Located at: model.input_value_map
value_map = nn.Linear(1, dim // 2)

# During forward pass:
calcium_features = model.input_value_map(input_values)
unit_features = model.unit_emb(input_unit_index)
input_tokens = torch.cat([calcium_features, unit_features], dim=-1)
```

### Input Sequence Construction

Unlike spike-based models, CaPOYO processes dense temporal grids:

```python
# For T timepoints and N neurons:
T, N = calcium_traces.df_over_f.shape

# Repeat timestamps for all neurons
input_timestamps = repeat(timestamps, "T -> (T N)", T=T, N=N)

# Flatten calcium values
input_values = rearrange(df_over_f, "T N -> (T N) 1")

# Repeat unit indices for all timepoints
input_unit_index = repeat(unit_indices, "N -> (T N)", T=T, N=N)
```

### Comparison with POYO+

| Component | POYO+ | CaPOYO |
|-----------|-------|--------|
| Input type | Sparse spikes | Dense calcium traces |
| Value mapping | None | `Linear(1, dim//2)` |
| Unit embedding dim | `dim` | `dim // 2` |
| Token types | Start/end/spike | Not used |
| Input shape | Variable (spike-dependent) | Fixed grid (T × N) |
| Temporal structure | Irregular spike times | Regular frame times |

## Use Cases

### Visual Coding Experiments
- Decoding visual stimulus properties from V1 responses
- Predicting behavioral states (running, pupil diameter)
- Multi-area decoding across visual hierarchy

### Calcium Imaging Analysis
- Population-level decoding from 2-photon imaging
- Multi-session learning across different imaging planes
- Transfer learning across different experimental conditions

### Multi-Task Learning
- Simultaneously predict stimulus and behavior
- Shared representations across related visual tasks
- Improved data efficiency through task relatedness
